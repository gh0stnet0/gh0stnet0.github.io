ZeroTier segmentation
isolated lab environments

A flat network is a liability. Segmentation isn't paranoia — it's the difference between a contained incident and a total compromise.

why flat networks fail

Most lab setups are flat. Everything on the same subnet, same broadcast domain, same trust level. It's convenient until it isn't. One misconfigured device becomes a pivot point into everything else.

ZeroTier gives you software-defined networking across physical boundaries. The segmentation logic lives in your rules, not your hardware. You can isolate a learner's environment from production infrastructure without touching a single physical switch.

the architecture

[operator node]
↕ full access
[ZeroTier controller]
↙ restricted            ↘ isolated
[lab segment A]         [lab segment B]
learner environment       production infra
✗ no cross-segment routing

The controller sees everything. Segments see only what you allow. A node in segment A cannot reach segment B regardless of what it tries. The rules enforce it, not trust.

setup — controller side

# install ZeroTier
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join <network_id>
 
# assign static IPs to managed nodes
# done via ZeroTier Central or self-hosted controller
# operator: 10.147.17.1
# lab node A: 10.147.17.50
# prod infra: 10.147.17.100-200

flow rules — the real segmentation

ZeroTier's flow rules are evaluated per packet. This is where isolation happens. Rules are defined in the network config and applied by every node.

# ZeroTier flow rules — network config
# allow operator to reach everything
if (ipDest 10.147.17.1/32) accept;
if (ipSrc 10.147.17.1/32) accept;
 
# isolate lab segment — no cross talk to prod range
if (ipSrc 10.147.17.50/32 and ipDest 10.147.17.100-200) drop;
if (ipDest 10.147.17.50/32 and ipSrc 10.147.17.100-200) drop;
 
# allow lab node internet access via default route
if (ipSrc 10.147.17.50/32) accept;
 
# default deny everything else
drop;

monitoring without access

The operator sees all traffic. The lab node sees nothing outside its segment. This asymmetry is intentional and useful — you can observe behavior in the isolated environment without the environment being able to observe back.

# monitor lab node activity from operator
tcpdump -i ztXXXXXX host 10.147.17.50
 
# check active ZeroTier peers
sudo zerotier-cli peers
 
→ 10.147.17.50 LEAF ONLINE 12ms
→ 10.147.17.103 LEAF ONLINE 8ms
practical use: this setup is ideal for onboarding environments. The learner gets a real network, real tools, real consequences — all contained. mistakes stay in the sandbox. you see everything. they see their segment.

what segmentation actually buys you

  • Blast radius control — incidents stay contained
  • Behavioral observation without interference
  • Credential isolation — lab creds don't touch prod
  • Clean rollback — nuke the segment, infra untouched
  • Auditability — every packet crossing segments is logged

A flat network trusts everything equally. That's not a network design — it's a liability.
